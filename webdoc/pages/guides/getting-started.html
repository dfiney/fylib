Primeiros Passos

Aprenda como integrar o fyLib ao seu projeto Angular e começar a criar interfaces incríveis em minutos.

1. Requisitos e Sintaxe Moderna

O fyLib v1.0+ requer Angular 17 ou superior. Todos os componentes internos foram migrados para a nova sintaxe de controle de fluxo do Angular, garantindo melhor performance e templates mais limpos.

Legado (v0.x)
<div *ngIf="show">
  <div *ngFor="let i of items">
    {{ i }}
  </div>
</div>
Moderno (v1.x+)
@if (show) {
  @for (i of items; track i) {
    {{ i }}
  }
}

2. Instalação Automatizada (Recomendado)

A forma mais rápida de começar é usando o comando ng add. Ele instala as dependências, cria os arquivos de configuração e prepara o bootstrapping da sua aplicação:

ng add @fylib/adapter-angular

O que o comando faz:

  • Instala @fylib/adapter-angular e módulos core (theme, config, etc).
  • Cria a pasta src/fylib/ com configurações padrão.
  • Configura o provideFyLib no seu app.config.ts.
  • Inicializa o FyLibService no app.component.ts.

3. Configuração via Provider (Precedência)

No seu arquivo app.config.ts, configure o serviço principal do fyLib usando o provideFyLib. Ao passar uma configuração aqui, ela se torna a fonte de verdade e o polling automático do JSON é desativado:

import { ApplicationConfig } from '@angular/core';
import { provideFyLib } from '@fylib/adapter-angular';
import { themeConfig } from './fylib/theme.config';
import { sseConfig } from './fylib/sse.config';

export const appConfig: ApplicationConfig = {
  providers: [
    provideFyLib({
      ...themeConfig,
      sse: sseConfig,
      logging: {
        enabled: true,
        level: 'debug',
        console: { enabled: true },
        localFiles: { enabled: true, path: 'fylogs' },
        remote: { enabled: false, endpoint: 'https://telemetry.example.com' }
      }
    })
  ]
};

A biblioteca reconhece automaticamente os arquivos como theme-controller.config.ts, sse.config.ts e crypto.config.ts quando eles são passados pelo provider, garantindo IntelliSense e tipagem forte.

4. Configuração Dinâmica (Fallback JSON)

Se o provideFyLib for utilizado sem argumentos (ou não for utilizado), o fyLib ativará automaticamente o monitoramento do arquivo remoto:

/fylib/theme-control/theme-controller.json

O polling ocorre a cada 3 segundos, permitindo trocar temas ou ativar efeitos globais instantaneamente através do JSON remoto. O provider sempre tem precedência sobre o JSON para evitar conflitos de configuração em tempo de execução.

5. Uso de Componentes

Importe os componentes desejados diretamente nos seus componentes standalone:

import { FyButtonComponent, FyCardComponent } from '@fylib/adapter-angular';

@Component({
  standalone: true,
  imports: [FyButtonComponent, FyCardComponent],
  template: `
    <fy-card title="Meu Primeiro Card">
      <p>Conteúdo do card aqui...</p>
      <fy-button variant="primary">Clique Aqui</fy-button>
    </fy-card>
  `
})
export class MyPageComponent {}
Nota sobre Ícones

O fyLib utiliza a biblioteca Phosphor Icons por padrão. Certifique-se de incluir o CSS do Phosphor no seu index.html para que os ícones renderizem corretamente.
