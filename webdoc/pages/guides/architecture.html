Arquitetura e Níveis de UI

Entenda os princípios técnicos que tornam o fyLib a plataforma de UI mais flexível e poderosa do mercado.

Estrutura de Módulos

O fyLib é construído sobre uma arquitetura de camadas desacopladas, garantindo que a definição do componente seja independente de sua implementação visual.

@fylib/catalog

O repositório de contratos. Define o "que" é o componente (props, variantes, features) sem conter HTML ou CSS.

@fylib/theme

Engine de Design Tokens. Controla toda a aparência visual através de variáveis dinâmicas e escalas de design.

@fylib/animation

Gerencia microinterações e efeitos globais (como corações ou confetes) de forma isolada do tema e do componente. Suporta parâmetros de intensidade, velocidade e loop.

Adapters (Angular/React)

A camada de renderização. Transforma os contratos do Catalog em componentes funcionais. O adapter Angular utiliza Angular 17+ com a nova sintaxe de controle de fluxo (@if, @for) para performance máxima. Seguro para SSR: Todos os componentes e diretivas realizam checks de plataforma (isPlatformBrowser) para garantir compatibilidade com Server-Side Rendering.

@fylib/logger

Sistema de logging centralizado. Suporta múltiplos destinos (console, arquivo remoto e local). No Node.js (SSR), grava diretamente em disco via FileSink, resolvendo caminhos relativos ou absolutos automaticamente.

@fylib/testing

Infraestrutura de testes baseada em Vitest e Angular Testing Library. Garante a confiabilidade do monorepo e a conformidade com padrões de Acessibilidade (A11y) WCAG 2.1 AA.

Os Três Níveis de UI

Dividimos nossos componentes em três níveis lógicos para garantir escalabilidade e orquestração eficiente.

01
Primitive Components

Blocos básicos de construção. Unidades visuais atômicas sem dependências externas.

Ex: Button, Input, Icon
02
Composite Components

Componentes de negócio que agrupam primitivos para formar unidades funcionais complexas.

Ex: Card, Accordion, Toast
03
Layout Structures

Unidades estruturais declarativas que orquestram o espaço e a responsividade global.

Ex: AppLayout, Grid, Sidebar

Hierarquia de Responsabilidades

  • Layout define a Estrutura.
  • Theme define a Aparência.
  • Animation define o Movimento.
  • Component define o Conteúdo.
